# 開発者説明書

## 目的
このプロジェクトを開発・改修する際の基本ルールをまとめる。
新機能の追加や既存機能の修正を行う場合は、以下の方針を必ず守ること。

## 開発ルール

### 1. 既存機能を壊さないこと
- 開発の際は、既存機能が壊れないように組み込むこと。
- 新しい処理を追加する場合は、既存の挙動に影響が出ないかを必ず確認すること。
- 既存コードを書き換える場合は、周辺機能への副作用を十分に考慮すること。

### 2. 新機能は新しい module で追加すること
- 新機能を実装する際は、既存の module に直接詰め込まず、新しい module として追加すること。
- 機能ごとに責務を分離し、保守しやすい構成を維持すること。
- UI、データ取得、状態管理など、役割が異なる処理はできるだけ分離すること。

### 3. 既存機能の修正は module から行うこと
- 既存機能を修正する場合は、その機能を担当している module を特定してから編集すること。
- 関係のないファイルや module に処理を追加しないこと。
- 一時的な回避ではなく、責務に沿った場所で修正すること。

## 推奨方針
- 変更前に、対象機能がどの module に属しているか確認すること。
- 追加実装より先に、既存構成との整合性を確認すること。
- module の責務が曖昧な場合は、無理に追記せず、分離を優先すること。

## まとめ
このプロジェクトでは、既存機能の安定性を最優先とする。
新機能は新しい module に追加し、既存機能の修正は既存 module の責務に従って行うこと。
